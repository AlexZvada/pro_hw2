false && true || true // true

Оскільки приорітет && вищий спочатку виконується він, результатом буде false, оскільки при логічному операторі && для результату true необхідно шоб обидва операнти були true

Далі йде порівняння false || true оскільки перший оперант false виконується другий true

false && (true || true) // false

Спочатку йде виконання true || true оскільки вони згруповані. Результатом буде true

Далі йде виконання false && true. Оскільки перший оперант логічного оператору І false другий не розглядається

+'40' + +'2'; // 42

Унарний + застосований до строки "40" перетворює її на число 40 так само унарний + застовоаний до строки "2" перетворює її на число 2.

40+2=42

'2' + 3 ** 2; // 29

Возведення в степінь має вищий приорітет за бінарний + тому спочатку 3 підносится в квадрат

при застосуванні бінарного + якщо один з операндів строка інший також перетворюється в строку

'2'+'9'=29

3 ** 2 / 3; // 3

** має вищий приорітет чим / тому спочатку 3 підноситься до квадрату, а потім ділиться на 3

3 ** (9 / 3); // 27

Спочатку виконуються згруповані операнди 9/3=3, далі 3 підноситься в куб

потім через бінарний мінус об'єднуються дві строки

Бінарний мінус має вищий приорітет тому спочатку строка '10' преобразовується в число 10-5=5

Потім йде порівняння, оскільки 5 не дорівнює 6 результат false

++number це скорочений запис від number = number + 1, тому спочатку перемінна number збільшується на 1, наступою строкою зменшується на 1
